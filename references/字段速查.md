# 字段速查（给小白 AI 一眼看懂）

> 目标：快速判断哪些字段可自动生成、哪些必须接口查询、哪些必须统一。

---

## 1) 第一张表（01_添加员工）

| 字段 | 来源 | 规则 |
|---|---|---|
| 姓名 | 自动生成 | 批次内唯一，不重复 |
| 手机号 | 自动生成 | 批次内唯一，11位手机号格式 |
| 企业名称 | 接口/登录态 | 必须来自当前登录公司 |
| 一级部门名称 | 接口查询 | 必须来自 `queryDepartments` |
| 二级部门 | 可空/规则生成 | 如无明确层级可空 |
| 备注 | 自动生成 | 可写“唯一性已校验” |

---

## 2) 第二张表（02_费用科目配置）

| 字段 | 来源 | 规则 |
|---|---|---|
| 一级费用科目 | 接口查询优先 | 来自 `queryFeeTemplate` 一级科目 |
| 二级费用科目 | 规则生成 | 与一级形成合理业务层级 |
| 三级费用科目 | 规则生成 | **全局不重复** |
| 归属单据类型 | 规则生成 | 仅：报销单/借款单/批量付款单/申请单 |
| 归属单据名称 | 规则生成（联动） | 必须与 Sheet3 `单据模板名称` 一一对应 |
| 单据适配人员 | 接口查询 | 来自 `queryCompany` 员工名 |
| 是否执行 | 固定 | 建议“是” |
| 备注 | 自动生成 | 可写规则说明 |

### 第二张表硬规则（必须）
1. 一级/二级可合并，三级不合并。
2. 每个二级科目下至少 2 个三级科目。
3. 三级科目全局不重复。
4. 同一 `归属单据名称` 下，`单据适配人员` 状态必须统一（全有或全无）。

---

## 3) 第三张表（03_单据表）

| 字段 | 来源 | 规则 |
|---|---|---|
| 单据分组（一级目录） | 规则生成 | 可启用向下继承（后续行留空） |
| 单据大类（二级目录） | 规则生成 | 仅四类：报销单/借款单/批量付款单/申请单 |
| 单据模板名称 | 联动生成 | 必须与 Sheet2 `归属单据名称` 一一对应 |
| 可见范围类型 | 规则生成 | 仅：角色/员工/部门 |
| 可见范围对象 | 接口查询随机 | 从真实角色/员工/部门池随机选 |
| 备注 | 自动生成 | 可写“来自真实接口” |

### 费用限制判断（给 Agent2 用）
- 仅报销单、批量付款单需要判断。
- 若 `归属单据名称` 非空 且 `单据适配人员` 非空 → 费用角色限制。
- 否则 → 直接限制末级费用科目。
- 借款单、申请单：不处理费用限制。

---

## 4) 必须接口查询的清单（不要手写）

1. 企业名称（登录态）
2. 部门列表（`queryDepartments`）
3. 员工列表（`queryCompany`）
4. 角色列表（`role/get/tree`）
5. 一级费用科目（`queryFeeTemplate`）
6. 审批流（`queryWorkFlow`）

---

## 5) 一键流程（最稳）

1. 先跑：`scripts/preflight_check.py`
2. 再跑：`scripts/generate_three_sheets_from_customer_template.py`
3. 验收：
   - Sheet2↔Sheet3 名称映射通过
   - Sheet2 层级/合并规则通过
   - Sheet3 可见范围对象来自真实接口
